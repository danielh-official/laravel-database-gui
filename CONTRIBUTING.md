# Contributing

We welcome contributions to the Laravel Database GUI project! Here are some guidelines to help you get started.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub. Make sure to include a clear description and steps to reproduce the issue.

## Pull Requests

1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Make your changes.
4. Ensure all tests pass.
5. Submit a pull request with a clear description of your changes.

## Coding Standards

Please follow the PSR-12 coding standard. You can use tools like PHP CS Fixer to automatically format your code.

## Testing

Run the test suite to ensure your changes do not break existing functionality:

```bash
composer test
```

Thank you for contributing!
